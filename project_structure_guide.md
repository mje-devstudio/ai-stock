# 프로젝트 구조 가이드

이 문서는 이 프로젝트의 디렉토리 구조와 각 디렉토리의 역할을 설명합니다.

## 전체 구조 개요

이 프로젝트는 키움증권 REST API를 활용한 자동매매 시스템으로, 텔레그램 봇을 통해 사용자와 상호작용하며 매매 전략을 실행합니다.

## 디렉토리 구조 및 역할

### `(최상위 폴더)`
**역할**: 메인 모듈이 위치한 디렉토리입니다.

프로그램의 시작점이 되는 메인 모듈이 포함됩니다. 메인 모듈은 프로그램의 전체 생명주기와 핵심 로직을 담당합니다.

### `api/`
**역할**: 키움증권 REST API 호출 관련 모듈들을 담는 디렉토리입니다.

키움증권 REST API를 호출하는 모듈들이 기능(TR)별로 포함됩니다.

### `realtime/`
**역할**: 실시간 데이터 수신 및 처리 관련 모듈들을 담는 디렉토리입니다.

웹소켓을 통한 실시간 데이터 수신 및 처리 로직이 포함됩니다.

### `telegram/`
**역할**: 텔레그램 봇 관련 기능을 담는 디렉토리입니다.

이 디렉토리는 텔레그램 봇의 핵심 기능을 구현합니다. 사용자로부터 받은 메시지를 파싱하고 명령어를 처리하는 모듈과, 텔레그램 메시지 전송 기능이 포함됩니다.

#### `telegram/commands/`
**역할**: 텔레그램 봇의 개별 명령어 처리 모듈들을 담는 디렉토리입니다.

각종 사용자 명령어를 처리하는 모듈들이 명령어별로 포함됩니다.

### `trading/`
**역할**: 매매 전략 및 로직을 담는 디렉토리입니다.

실제 매매를 수행하는 전략 로직들이 포함됩니다. 다양한 매매 전략이 모듈별로 구현됩니다.

### `utils/`
**역할**: 공통 유틸리티 함수 및 헬퍼 모듈들을 담는 디렉토리입니다.

프로젝트 전반에서 사용되는 공통 기능들이 포함됩니다.

### `config/`
**역할**: 설정 파일 및 인증 정보를 저장하는 디렉토리입니다.

프로그램 실행에 필요한 설정 파일들이 저장됩니다.

#### `config/data/`
**역할**: 프로그램 실행 중 생성되는 동적 설정 파일들을 저장하는 디렉토리입니다.

프로그램 실행 중 생성되고 변경되는 데이터 파일들이 JSON 형식으로 저장됩니다.
단순한 형태의 설정들은 해당 디렉토리의 `settings.json` 문서에,
해당 설정들의 디폴트 값들은 `settings-default.json` 문서에 저장됩니다.

### `tr_docs/`
**역할**: 키움 REST API의 각 기능(TR)에 대한 명세 파일들을 담는 디렉토리입니다.

키움증권 REST API의 각 기능에 대한 명세가 파일들로 저장됩니다.
